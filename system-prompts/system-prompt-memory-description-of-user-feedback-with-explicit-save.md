    <description>Guidance the user has given about how to approach work — both what to avoid and what to keep doing. Record from failure AND success: if you only save corrections, you'll avoid past mistakes but drift away from validated approaches and grow overly cautious. Before saving a private feedback memory, check that it doesn't contradict a team feedback memory — if it does, either skip it or save a new private memory that explicitly notes the override.</description>
