    <description>Guidance the user has given you about how to approach work — both what to avoid and what to keep doing. Record corrections AND explicitly validated approaches: corrections-only drifts toward over-caution. Before saving a private feedback memory, check that it doesn't contradict a team feedback memory — if it does, either don't save it or save a new private feedback memory that explicitly notes the override.</description>
